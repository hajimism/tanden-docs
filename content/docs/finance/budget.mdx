Tandenの予算は**プロジェクトファイナンス**の考え方をベースに、[Warifuコード](/docs/tools/warifu)を最小単位として管理します。コードは[Warifuフォルダ](/docs/tools/warifu)で入れ子に束ねられ、すべてのお金はWarifuコードを通じて流れます。

## 全体構造

| 区分 | 仕組み | 原資 |
|---|---|---|
| **Warifuコード** | プロジェクト単位の予算管理 | 委譲された予算 |
| **レーベルP&L** | レーベル配下のコードの集計 | レーベルの売上 |
| **スタジオ予算** | 全社一括 | 全社売上の固定%（目安15〜20%） |
| **Basic プール** | 全社一括 | 全社予算から確保 |
| **学習プール** | 全社一括 | 全社予算から確保 |

## 予算の委譲

予算は社長を起点に、人から人へ委譲されます。

- 誰でも自分が持つ予算の一部を他者に委譲できます
- 委譲された予算で[Warifuコード](/docs/tools/warifu)を発行するもよし、さらに誰かに再委譲するもよし
- **社長のみ**、任意の予算を強制的に回収する権限を持ちます

```
社長（全社予算）
  ├─ レーベル主宰Aに委譲
  │    ├─ Acme案件/
  │    │    ├─ PJT-ACMQ1（コード、Q1予算）
  │    │    └─ PJT-ACMQ2（コード、Q2予算）
  │    ├─ バンドリーダーBに再委譲
  │    │    └─ 新業態実験/
  │    │         └─ PJT-XYZ001（コード）
  │    └─ レーベル運営/
  │         └─ OPS-LAB001（コード）
  └─ スタジオに委譲
       └─ 社内ツール開発/
            └─ STD-DEV001（コード）
```

## Warifuコードの2つのクラス

prefixでクラスが決まります。

| クラス | 意味 | 売上の有無 | 主要指標 | prefix例 |
|---|---|---|---|---|
| **Revenue（売上型）** | クライアントがいる。売上が立つ | あり | 利益率 | `PJT` |
| **Budget（投資型）** | 社内投資。予算を使い切るのがデフォルト | なし | 消化率・残額 | `OPS`, `WCL`, `LRN`, `HIR`, `STD` |

- Revenue型は四半期など計測単位で区切って発行します。長期案件はフォルダの中に期間別コードを並べて集計します
- フォルダ作成とコード発行は誰でもできますが、コードに予算をつけるのは予算を持つ人だけです
- 新しいprefixを追加する際に、どちらのクラスかを指定します

コードの形式・ライフサイクルなど詳細は [Warifu（割符）](/docs/tools/warifu) を参照してください。

### イケてる／ダサい

**Revenue型：** 利益率は高ければいいというものではなく、低くても罰せられません。利益率はプロジェクトの健全性を測るための指標であって、最大化するための数字ではありません。全社の[Karute](/docs/tools/karute)で健全なガバナンスが取れていることを全社で目指します。

**Budget型：** 少ない予算で大きな効果を出せたらかっこいい。予算は使い切ることがデフォルトですが、投資対効果を意識するのはイケてます。

**予算の委譲：** 予算を任されることは信頼の証です。かっこいい。「予算を持っているから偉い」と振る舞うのはダサい。

## レーベル会計

レーベル自体が最上位のWarifuフォルダとして機能します。レーベルP&Lは、そのフォルダ配下の全コードの集計です。

```
レーベル売上 = Σ Revenue型コードの売上
レーベルコスト = Σ 全コードのコスト（人件費 + 経費）
レーベル利益 = 売上 - コスト
```

- レーベル内の[バンド](/docs/org-design/band)への予算配分はレーベルの裁量です
- [ボーナス](/docs/finance/salary)の原資もレーベルのP&Lに連動します
- レーベルの[Karute](/docs/tools/karute)で財務の健全性が可視化されます

## スタジオ予算

[スタジオ](/docs/org-design/studio)は全社売上の固定%（目安15〜20%）で予算を確保します。特定レーベルの業績に依存しないことで、全レーベルに対して中立的な基盤を提供できます。

### CDレーベルからのキックバック

[CDレーベル](/docs/saas)のSaaS売上の一部（目安30%）はスタジオに「プロダクト利用料」としてキックバックされます。スタジオが作ったプロダクトをCDレーベルが外部に届けて得た売上なので、開発元に還元する構造です。

- スタジオにとっては「いいプロダクトを作る → CDレーベル経由でSaaS売上が伸びる → 自分たちの予算が増える」インセンティブになります
- CDレーベルにとっては「プロダクト改善の要望を出すことが自分たちの投資になる」構造です
- 初期はSaaS売上が小さいため、スタジオは固定%だけで運営し、売上が立ち始めたらキックバックを導入します

## 新卒支援金

[レーベル](/docs/org-design/label)が新卒を採用する際、全社予算から新卒支援金を支給します。新卒は即戦力ではないため、育成期間のコストをレーベルのP&Lだけで負担すると採用のハードルが上がります。

- 新卒1人あたり定額で支給（入社から6ヶ月間の人件費の一部をカバー）
- 支給元は全社予算（スタジオ予算の固定%とは別枠）
- レーベルが新卒を採るハードルを下げつつ、選ぶ責任はレーベルに残します

## Basic プール

AIやGitHubなど「人権」的なツールは、会社が別途予算を取って全員に提供します。[Meguriポイント](/docs/finance/meguri)を消費する必要はありません。目安は月額1〜1.5万円/人程度です（Google Workspace、Slack、Notion、GitHub、AI等）。

## 学習プール（ファンディング）

全社的にプールしている学習予算です。個人のMeguriポイントでは足りない投資をしたいとき、ファンディングテイアン＋LGTMで学習プールから予算を取れます。取得した予算はMeguriを通じて個人予算に付け替えられ、[Motode](/docs/tools/motode)で消費します。
