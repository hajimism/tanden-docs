## アサインの3つの経路

プロジェクトへの参加は、単一の仕組みに依存しません。複数の経路が自然に機能する状態を目指しています。

### 1. 手挙げ（Noroshi）

プロジェクトごとにジョブディスクリプションを徹底的に言語化し、[Noroshi（ジョブボード）](/docs/tools/noroshi)に掲載します。興味のある人が自分で手を挙げます。

### 2. 声がけ（関係性ネットワーク）

日々の業務やチームビルディングで築いた関係性の中から、「この人に合いそう」と声をかけることも活発です。[マイコーチ](/docs/systems/my-coach)からの推薦もこの経路に含まれます。

### 3. AIマッチング（Shiorie）

[Shiorie](/docs/tools/shiorie)が[魂のカタチ](/docs/culture/values/soul-shape)やスキルセットをもとに、マッチしそうな人にリマインドを送ります。Noroshiに掲載されたJDと個人のプロフィールを照合し、気づきを促す仕組みです。

## 同時に持てるプロジェクト数

上限はありません。自分の状態と相談して、本人が判断します。

力が湧いているかどうかは[Tsuzuri](/docs/tools/tsudzuri)で日々開示されるので、周囲も状態を見に行けます。

## プロジェクトのコスト管理

プロジェクトのコストは[Warifuコード](/docs/tools/warifu)で管理します。

- プロジェクトごとにWarifuコードが発行され、予算が紐づきます
- メンバーは[Kodou（鼓動）](/docs/tools/kodou)で「このコードに何時間」と記録します。[社内単価](#社内単価) × 時間で人件費がコードにチャージされます
- プロジェクトに関わる経費は[Motode](/docs/tools/motode)からコードを指定してチャージします
- メンバーが意識するのは「どのコードに鼓動を刻むか」だけです。予算管理はコードの責任者が行います

### レーベル横断プロジェクト

[レーベル](/docs/org-design/label)をまたぐプロジェクトでは、コード発行時にどのレーベルのP&Lに帰属させるかを決めます。コストを複数レーベルで按分する場合は、レーベルごとに別コードを発行してメンバーが適切なコードに鼓動を刻みます。

## 社内単価

社内単価は[ベース年収](/docs/finance/salary)から自動算出されます。

```
社内単価（時給） = ベース年収 × 1.3 ÷ 1,920時間
```

| 要素 | 値 | 備考 |
|---|---|---|
| ベース年収 | 本人のベース年収 | [ステージ](/docs/finance/salary#ステージ制)のレンジ内で決定済み |
| 係数 1.3 | 法定福利費＋間接コスト | 社会保険料の会社負担分等を含む |
| 1,920時間 | 年間基準時間 | 月160時間 × 12ヶ月 |

| ステージ | 年収レンジ | 社内単価レンジ |
|---|---|---|
| S1 | 500〜700万円 | 約3,400〜4,700円/h |
| S2 | 600〜1,000万円 | 約4,100〜6,800円/h |
| S3 | 850〜1,400万円 | 約5,800〜9,500円/h |
| S4 | 1,200〜2,000万円 | 約8,100〜13,500円/h |

- 社内単価は全社公開です。「自分が1時間このプロジェクトに入ると、コードにいくらチャージされるか」が常に見えます
- 給与テイアンが通れば単価も自動で更新されます。別の承認フローは不要です
- 自分のコストを知ること自体がコンテキストデザインです。「この仕事に自分が入る価値があるか」を本人が判断できます
