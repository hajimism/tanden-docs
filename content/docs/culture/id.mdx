## I/Dとは

**I**ケてる・**D**サい。略してI/D。

何をイケてるとし、何をダサいとするか——その集合が、その人の**Identity**になります。ルールは「やるな」と言いますが、I/Dは「こっちの方がかっこいいよね」と誘います。Tandenでは、ルールを最小にする代わりに、I/Dを全社のコンテキストとして共有します。

ダサいは「ダメ」ではありません。自分の中のダサさに気づいていること、それでも今はこっちを選んでいると自覚していること——その誠実さ自体がイケてます。ダサい自分を愛してもいいのです。禁止ではなく、鏡です。

以下はTandenが全社的に「イケてる」「ダサい」と考えている振る舞いの一覧です。最終的に自分にとって何がイケてるかは各自が決めます。自分のI/Dを言語化し、表明すること自体がコンテキストデザインです。

## 自律と判断

| イケてる | ダサい |
|---|---|
| コンテキストを受け取って自分で判断する | 指示を待つ。判断を人に委ねる |
| 原則を持った上で使い分ける | 「とにかく〇〇しとけ」で思考停止する |
| 原則を持った上で使い分ける | 「場合による」で止まる |
| 違和感を覚えたらテイアンで変えにいく | 従うだけか、陰で不満を溜めるだけ |
| 不可逆な判断は丁寧に、可逆な判断は速く | 全部同じ重さで悩む |

## 言語化と透明性

| イケてる | ダサい |
|---|---|
| 判断・提案・学び・失敗をドキュメントにして残す | 口頭で済ませて消える |
| 聞かれる前に背景・意図・制約を共有する | 聞かれてから答える |
| 自分のPrinciplesとI/Dを言語化して公開する | 考えはあるけど表明しない |
| 自分の状態を隠さず開示する | 力が枯れているのに黙って耐える |
| 経費も数字もオープンにする | 見せたくない情報を隠す |

## 提案と意思決定

| イケてる | ダサい |
|---|---|
| 材料を揃えて、相手が判断できる状態をつくってから提案する | 材料なしに「こうしてほしい」と言う |
| 実行しやすいコンテキストをデザインした上でテイアンする | 「テイアンしたのにやってくれない」と不満を言う |
| 自分の価値を材料揃えて堂々と言語化する（給与テイアン） | 振る舞いを棚に上げて金額だけ主張する |
| テイアンで制度を変えにいく | 制度への不満を飲み込んで従う |

## フィードバックと対話

| イケてる | ダサい |
|---|---|
| 気づいたことを直接本人に伝える | 陰で「あの人ダメだよね」と言う |
| 褒めはオープンに、お叱りはこっそりと | 公開の場で人を詰める |
| 対立を対話に変える。第三者を巻き込む | 対立を避けるか、平行線のまま放置する |
| パフォーマンスの問題を感じたら、まず自分が何をしていないか振り返る | 本人だけの問題にする |

## スピードと実行

| イケてる | ダサい |
|---|---|
| 60点を速く出してフィードバックで磨く | 完璧に仕上がるまで抱え込む |
| MTGは決めるべきことを決めたら終わる | 時間いっぱい使い切る |
| 走りながら考える。計画は仮説として扱う | 計画通りに進めることが目的化する |
| 必要な投資を自分の判断で素早く行う | 使える手段やコンテキストがあるのに知ろうとしない |

## 人との向き合い方

| イケてる | ダサい |
|---|---|
| 仕事を渡す前に相手の魂のカタチを見に行く | 空いてる人にとりあえず振る |
| 相手の力が湧く仕事を一緒に考える | 効率だけでアサインする |
| 他者の学びを促進する。知見を使える形にして渡す | 自分の中だけで完結させる |
| 互いのI/Dの違いをリスペクトする | 自分のI/Dを押しつける |

## 組織と役割

| イケてる | ダサい |
|---|---|
| 自分も何かを賭けている。矢面に立つ | 影響力だけ行使してリスクを取らない |
| 先回りしてインフラを整える。自分たちからも企みを仕掛ける（スタジオ） | 言われたことだけ処理する御用聞き。ルールで支配する門番 |
| 歓びの密度で競う。「あのレーベル、楽しそう。悔しい」 | 数字のランキングに堕ちる |
| 管理的な体質を魂のカタチとしてリスペクトする | 「管理はダメ」と言って人の力を抑え込む |

## 学びと成長

| イケてる | ダサい |
|---|---|
| 広さで深さを加速させる。軸と関係ある広さに投資する | 軸と無関係に手を広げて散漫になる |
| 自分で選んだ規律を守る。それが自由を拡張する | 他人の不安を埋めるためだけの規律に従う |
| 成熟の三段階を登る。感覚で選べるまで磨く | 「これが最強」で固定する |
| シオリエに文脈を引き出させた上で自分で判断する | 「シオリエが言ったからやる」と判断を委ねる |

## あなたのI/Dを書く

上記はTandenの全社I/Dです。これに加えて、**自分だけのI/D**を言語化してください。

- 仕事の進め方、コミュニケーション、美意識、こだわり——何でもOKです
- [ojtチャンネル](/docs/work-style/ojt-channel)に投稿し、[Meibo](/docs/tools/meibo)に登録します
- 他の人のI/Dを読むことで、相手の魂のカタチが見えてきます
- I/Dは変わっていいものです。成熟するほど更新されます

自分のI/Dを持つことは、[Principles（判断の原則）](/docs/culture/stance)と対をなす行為です。Principlesが論理の軸なら、I/Dは美意識の軸。両方が揃ったとき、あなたがどんなコンテキストデザイナーかが周囲に伝わります。
