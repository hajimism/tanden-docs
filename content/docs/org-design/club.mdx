## クラブとは

レーベルやバンドを超えて、関心や職能でゆるく集まるコミュニティです。Slackチャンネルでの日常的なやりとりから、定期的な勉強会、山登りサークルまで——粒度は問いません。

## なぜクラブが必要か

Tandenの組織はレーベル・バンドを骨格としたツリー構造ですが、[実態はセミラティス](/docs/org-design)です。ツリーの骨格だけだと、横の線が足りません。

たとえば、レーベルにエンジニアが一人だけだったら。バンドの中では「エンジニアの視点」を頼られる存在ですが、同じ専門性を持つ仲間がいない孤独感があります。エンジニアクラブがあれば、レーベルを超えて技術の壁打ちができ、スキルアップのきっかけも生まれます。

友達を増やしてほしいのです。

## クラブの種類

いろいろな粒度で存在していてOKです。

| 種類 | 例 |
|---|---|
| **職能クラブ** | エンジニアクラブ、デザインクラブ、ライティングクラブ |
| **関心クラブ** | AIクラブ、組織論クラブ、教育クラブ |
| **趣味クラブ** | 登山クラブ、読書クラブ、ボードゲームクラブ |

常設でも、目的を果たしたら解散する時限でも構いません。

## 始め方

**誰でも、いつでも**始められます。テイアンは不要です。

1. Slackチャンネルを作る
2. [Meibo](/docs/tools/meibo)のGroupsに登録する
3. 人を誘う

それだけです。

## 予算

クラブに専用の予算枠はありません。人が集まって対話すること自体に予算は要りません。

勉強会で外部講師を呼びたい、合宿をしたいなど、学びに繋がる活動であれば[学習プール](/docs/finance/budget)へのファンディングテイアンで対応できます。

## クラブとバンドの違い

| | バンド | クラブ |
|---|---|---|
| **目的** | 成果を出す | つながる・学ぶ・楽しむ |
| **人数** | 4〜6人 | 制限なし |
| **所属** | レーベル内 | レーベル横断 |
| **責任** | プロジェクトの成果 | なし |
| **予算** | レーベル予算 | なし（テイアンで都度） |
