## 設計思想

組織は管理の道具ではなく、コンテキストです。

管理の道具としての組織は「間違えるな」と言います。コンテキストとしての組織は「一緒に企もうぜ」と言います。Tandenが目指すのは後者——全員がコンテキストデザイナーとして企み続けられる組織です。

スケーリングの問いは「どうやって1000人を管理するか」ではなく、**「どうやって1000人が企み続けられるコンテキストをつくるか」**です。

## 全体構造

Tandenの組織は、音楽業界のメタファーで構成されています。

| 概念 | 役割 |
|---|---|
| **レーベル** | 事業領域の束。複数のバンドとプロデューサーチームを持つ |
| **バンド** | 4〜6人の少数精鋭チーム。企みの最前線 |
| **プロデューサー** | レーベル内でバンドの企みを増幅するチーム |
| **スタジオ** | 全レーベル共通の基盤。人事・財務・経営企画・法務のグループ |

レーベルの一つである[CDレーベル](/docs/saas)（コンテキストデザインレーベル）は、スタジオが開発したプロダクトを外部の組織に届ける事業です。スタジオが作り、CDレーベルが届けます。

```
社長
 │
 ├── レーベルA（プロデューサーチーム + バンド群）
 ├── レーベルB（プロデューサーチーム + バンド群）
 ├── CDレーベル（プロデューサーチーム + 顧客別バンド群）⇄ スタジオ
 │
 └── スタジオ（人事・財務・経営企画・法務グループ）
```

## ツリーではなくセミラティス

組織図はツリー（木構造）に見えますが、実態はセミラティス（半束構造）です。一人が複数のコンテキストに同時に属しています。

- 所属ユニットでの日常業務
- ユニットを越えた越境プロジェクト
- マイコーチ関係による斜めの対話
- 関心ベースの横のネットワーク

レーベルやバンドはツリーの**骨格**です。実際の組織は、その間を走る無数の横の線・斜めの線で構成されています。

## バンド・スタジオ・プロデューサーは「箱」ではなく「モード」

レーベル・バンド・スタジオは固定の所属ではなく、振る舞いのモードです。ある人の1週間が月〜水はバンドモード（現場で企む）、木はスタジオモード（仕組みの改善）、金はプロデューサーモード（別バンドの壁打ち）ということもありえます。

もちろん専任が合う人もいます。経理が魂のカタチの人は週5スタジオモードで最高に力が湧くかもしれません。でもそれは強制ではなく、魂のカタチで自然にそうなるものです。

## スケーリング

4〜6人のチームを基本単位にすると、4〜5階層で1000人規模に到達します。社長が定期的に対話するのは上位2階層の20〜42人です。

- **広く薄く**: 社長はSlackで全社員に可視。誰でもメンションで直接話せます
- **狭く深く**: 主要なコンテキストデザイナー（プロデューサーチームのリーダー層）としっかり時間を取ります

社員側から社長が見えていることが大切です。社長が楽しそうに企んでいる姿が流れてくるだけで、コンテキストとして機能します。

## バンドは天井を上げ、スタジオは床を上げる

バンドの成果は「このバンドだからこそ出せた」突出です。魂のカタチと鍵穴が噛み合った瞬間の爆発力。再現性がなくていい。

スタジオの成果は「誰がやっても、どのレーベルでも、この水準は出る」という底上げです。仕組みの力でベースラインを上げていきます。再現性こそが価値です。

床が高いからバンドの尖りが「土台の上での飛躍」になり、天井を突き破った事例があるからスタジオは「次はこの水準を床にしよう」と仕組み化の目標が見えます。
