## 原則

Tandenのコンテンツ戦略は**「コンテンツマーケティング」という活動を持たないこと**です。

経営を透明にやる → 透明にやっていること自体がコンテンツになる → 人が集まる。この順番を守ります。

### 設計の前提

- **中の人がまず面白いと思えるものだけ出す。** 社内の誰も読みたくないコンテンツを外に出しても刺さらない
- **編集コストが低い。** 元の素材が社内にすでにある状態で、編集・公開するだけ
- **続けるのが苦じゃない。** 業務の一部だから。「マーケのために書く」コンテンツは続かない
- **嘘がつけない。** リアルだから面白い

### やらないこと

- マーケ担当を雇って記事を量産する
- SEOキーワードから逆算してブログを書く
- 「自律型組織 メリット」のような一般論記事
- 社内の誰も読まない企業広報

## コンテンツ一覧

### Open Handbook

このサイトそのものです。制度が変わるたびにコンテンツが自然に増えます。[テイアン](/docs/systems/teian)で制度が変わる → Handbookに反映 → 差分が記事の種になる。

| 効果 | |
|---|---|
| 社内 | 新入社員のオンボーディング教材。全社員のリファレンス |
| 採用 | 「入る前から全部わかる」安心感 |
| SaaS | ツールの使い方と思想がセットで伝わる。導入マニュアルを兼ねる |

### 社長note

代表が経営判断の裏側を書きます。何を考えてこう決めたか——意思決定のコンテキストそのもの。

| 効果 | |
|---|---|
| 社内 | 代表の思考が全社員に共有される。コンテキストの流通そのもの |
| 採用・SaaS | 思想に共感する人を惹きつけるファネルの入口 |
| 業界 | 「コンテキストマネジメント」という言葉を市場に定着させる |

### 社内インタビュー

[CDレーベル](/docs/saas)の[バンド](/docs/org-design/band#例cdレーベルのバンド)メンバーがTandenの中の人を取材し、**力が湧いている人の姿**を伝えます。

| 効果 | |
|---|---|
| 社内 | インタビューされる側が自分の「魂のカタチ」を言語化する機会になる。社内の相互理解が深まる |
| 採用 | 「この会社にはこういう人がいる」が最強の採用コンテンツ。求人票より人の顔が見える |
| SaaS | 「Tanden Toolkitを使ってる会社の人がこんなに活き活きしてる」→ 機能紹介より説得力がある |
| CDレーベル自身 | 社内を横断して取材することで、クライアントに伝えるTandenのリアルが常に更新される |

ツールの機能説明を一切しなくても、人の話の中に自然にツールが出てきます。「Okuriでこんなフィードバックもらって嬉しかった」「Teianで制度変えたんですよ」——これが最強のプロダクトマーケティングです。

**体制：** CDレーベルのバンドメンバーが業務として制作します。クライアント伴走と社内取材を同じ人が担うことで、「Tandenの今」を知っている人がコンテンツを作り、「コンテンツを作る過程で得た文脈」がクライアント伴走の質を上げる——この循環がCDレーベルの[バンドチーム設計](/docs/org-design/band#バンドチーム一つの脳みそになる)の核心です。

### マチネの公開

毎週金曜の[マチネ](/docs/work-style/matinee)（社内勉強会）を録画・公開します。

| 効果 | |
|---|---|
| 社内 | 欠席者のアーカイブ。過去の発表を新人が追える |
| 採用 | 「この会社の知的水準」が一発でわかる。面接より雄弁 |
| SaaS | 「この人たちに組織を見てもらいたい」と思わせる |
| 登壇 | 外部カンファレンスのプロポーザルにそのまま出せる |

元から面白いと思ってやっている勉強会を公開するだけ。追加コストはほぼゼロです。

### 組織のバグレポート

うまくいかなかったこと、制度が壊れたこと、判断を間違えたことを正直に書きます。

| 効果 | |
|---|---|
| 社内 | ポストモーテムとして再発防止になる。「うちはこれで失敗した」が共有資産になる |
| 採用 | 失敗を隠さない文化が伝わる |
| SaaS | 「承認なし」で起きた問題とその対処を見せることで、導入不安を先に潰せる |
| 信頼性 | 成功談だけの会社は胡散臭い。失敗を出すことでOpen Handbookの信頼度が上がる |

例：
- テイアンで決まったけど3ヶ月で撤回した制度の話
- Warifuコードの設計をミスって予算が溶けた話
- 「魂のカタチを活かす」を誤解して起きたこと

### 給与テイアンの実況

実際の[給与テイアン](/docs/finance/salary)の準備→提出→レビュー→決定のプロセスを匿名で公開します。

| 効果 | |
|---|---|
| 社内 | 次に給与テイアンを出す人の参考資料。「こうやって材料を揃えればいいのか」がわかる |
| 採用 | 「自分で給与を提案する」の実態が見える。怖さが消える |
| SaaS | [Teian](/docs/tools/teian)の実際の使われ方が伝わる |
| メディア | 「社員が自分で給与を決める会社」は取材されやすい |

社内で最も読まれるコンテンツになりえます。他の人がどう給与テイアンを組み立てたかは、誰もが気になる情報です。

### 企みの実況中継

誰かが新しいことを始めるとき（新プロジェクト、新制度、新ツール）、そのプロセスをリアルタイムで公開します。

| 効果 | |
|---|---|
| 社内 | 他の人の企みから学べる。「あのプロジェクトどうなってるの？」が解消される |
| 採用 | 「入社したらこういう企みができるのか」が伝わる |
| SaaS | コンテキストマネジメントが動いている様子そのもの |
| 教育 | 「自律的に仕事を進めるとはどういうことか」の生きた教材 |

完了後の振り返りではなく**進行中に出す**のがポイントです。結果がわからない状態で公開するから面白い。

### 経営数字の定点観測

[Karute](/docs/tools/karute)の数字を四半期ごとに公開します。売上、利益率、Meguriポイントの循環量、Okuriの送信数、テイアンの件数。

| 効果 | |
|---|---|
| 社内 | 全社の状態を定期的に振り返る機会 |
| 採用 | 「入る前から会社の財務が見える」は他社にない |
| SaaS | Karuteでこういう指標が見えるというライブデモ |
| 信頼 | Bufferの透明性レポートの日本版。これだけで話題になる |

### ポッドキャスト

代表が他の経営者と対話します。「コンテキストマネジメント」の言葉を広める場です。

| 効果 | |
|---|---|
| 社内 | 代表の考えが対話を通じて深まる。社員も聴ける |
| 採用・SaaS | 1人で書くより対話の方が思想が伝播しやすい |
| ネットワーク | ゲストとの関係構築。ゲストの会社がクライアント候補になることもある |

## 体制

コンテンツ制作は[CDレーベル](/docs/saas)の業務に内包されます。「マーケ担当」を別に置くのではなく、クライアント伴走をしている人がそのまま書きます。

| フェーズ | 体制 | やること |
|---|---|---|
| プレ起業 | 代表1人 | Open Handbook公開、note執筆、反応を見る |
| フェーズ1 | 代表 + CDレーベルのバンド | note継続、社内インタビュー開始、ポッドキャスト開始。バンドメンバーがクライアント伴走と並行してコンテンツを制作 |
| フェーズ2 | 代表 + CDレーベルのバンド | マチネ公開、バグレポート、企みの実況、透明性レポート追加。導入企業のBefore/Afterは伴走したメンバー自身が書く |
| フェーズ3以降 | 変えない | コンテンツは経営の副産物。チームを増やす必要がない |

フェーズ3でも増やさなくていい理由：

- **顧客が増えるほどコンテンツが増える。** 導入事例、Karuteのデータ、Teianのログ。素材は勝手に増える
- **顧客自身がコンテンツになる。** 「うちもTanden Toolkit入れました」を顧客が発信する
- **M&Aした会社もコンテンツになる。** 「買収されたけど何も変わらなかった。むしろ良くなった」は強烈なストーリー
- **CDレーベルのバンドが増えるほど書ける人が増える。** コンテンツ専任を増やすのではなく、伴走する人が書くから自然にスケールする

マーケの人数は増やさず、**コンテンツの発生源を増やす**のがTandenらしい成長です。

## 「第二の声」を育てる

[創業者依存](/docs/saas/strategy)を緩和するために、代表以外の発信者を早期から育てます。

CDレーベルの誰かやスタジオのエンジニアが、自分の言葉で外に発信する文化を初期から作ります。[ojtチャンネル](/docs/work-style/ojt-channel)の仕組みがそのまま使えます。社内で書いている人の中から「この人の文章、外にも出したい」を見つけて、本人の意思で公開に移行する流れです。
