## ポジショニング

**Tanden Toolkit は「コンテキストマネジメント」を実装するSaaS。**

機能を売るプロダクトではありません。「承認なし・自律分散・コンテキストを流通させる経営」というスタイルを、ツールの形で届けます。競合はHR SaaSや経費精算ツールではなく、**経営コンサルティング + それを実装するソフトウェア**の組み合わせです。

## ファネル

```
Open Handbook（無料公開）
  ↓ 思想に共感
カルチャーツール（Teian, Okuri, Tsuzuri, Meguri）
  ↓ 組織運営に踏み込む
ファイナンスツール（Warifu, Kodou, Motode, Karute）
  ↓ コミュニケーション基盤を移行
Uneri + Shiorie（チャット + AI）
  ↓ 完全移行
フルスイート + 組織コンサルティング
```

Open Handbookがファネルの入口です。ハンドブックを読んで「うちもやりたい」と思った企業がクライアントになります。広告費ゼロのコンテンツ・レッド・グロース。

## ロードマップ

### フェーズ1：PMF検証（0-12ヶ月）

Meibo + Teian + Okuri + Tsuzuri + Meguri をSlack連携で提供します。

- 開発コストが低く、PMF検証が速い
- Open Handbook経由で思想共感層を獲得
- Slack上でShiorieをボットとして提供（Uneriの前段階）
- 目標：10社導入、DAU70%以上、月次チャーン3%未満

### フェーズ2：統合の価値を証明（12-24ヶ月）

Warifu + Kodou + Motode + Karute を追加します。

- プロジェクトファイナンスの統合がキラーフィーチャー
- 「バラバラのSaaSよりTanden Toolkitで統合した方がいい」が明確になる
- 既存顧客のアップセル（ツール追加）でNRR120%以上を目指す
- 目標：50社、MRR500万円、平均6ツール/社

### フェーズ3：プラットフォーム化（24-36ヶ月）

Uneri + Soubi + Noroshi を追加し、Slack依存を解消します。

- Shiorieのネイティブ統合で体験が劇的に変わる
- 既存顧客のSlack→Uneri移行を促す
- フルスイート導入率30%以上を目指す
- 目標：150社、MRR2,000万円

## 獲得戦略

| チャネル | 施策 |
|---|---|
| **Open Handbook** | SEO + SNS拡散。「自律型組織」「承認なし」「コンテキストマネジメント」等で検索流入 |
| **代表の発信** | 経営思想の発信（note、登壇、ポッドキャスト）。思想で惹きつけ、ツールで刈り取る |
| **事例公開** | 導入企業のBefore/Afterを詳細に公開 |
| **コミュニティ** | コンテキストマネジメントに関心のある経営者コミュニティ |
| **CDレーベルの伴走** | 導入初期の組織コンサルティングが口コミを生む |

**やらないこと：** マス広告、テレアポ、機能比較での競争。

## 競争優位性

| 優位性 | 説明 |
|---|---|
| **思想のロックイン** | 機能ではなく経営スタイルが刺さるので、他ツールに乗り換えにくい |
| **14ツールの深い統合** | データが途切れなく流れる。バラバラのSaaSでは再現不可能 |
| **Open Handbook** | 導入マニュアルが公開されており、CSコストが低い |
| **Shiorie** | ツール横断AIは既存SaaSが後付けでは実現しにくい |
| **Cloudflareスタック** | インフラコストが売上の1%未満。価格競争力が高い |

## KPI

| フェーズ | 主要KPI | 目標 |
|---|---|---|
| フェーズ1 | 導入社数、DAU、チャーン率 | 10社、DAU70%+、チャーン＜3% |
| フェーズ2 | MRR、導入ツール数/社、NRR | MRR500万、平均6ツール、NRR120%+ |
| フェーズ3 | MRR、フルスイート率 | MRR2,000万、フルスイート30%+ |

**NRR（Net Revenue Retention）が最重要指標。** ツール追加によるアップセルが自然に起きる設計なので、NRR120%以上を維持します。

## リスクと対策

| リスク | 対策 |
|---|---|
| 市場が狭い | 「自律型組織」に限定せず「透明性のある経営」まで広げれば対象は増える |
| 14ツールの開発が重い | フェーズ分割で最小限から。カルチャーツールは開発が軽い |
| Slack移行ハードル | フェーズ1-2はSlack共存。Uneriは十分な価値が溜まってから |
| 文化が合わず解約 | 導入前にOpen Handbookで思想の合意を取り、ミスマッチを減らす |
