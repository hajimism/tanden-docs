## ポジショニング

**Tanden Toolkit は「コンテキストマネジメント」を実装するSaaS。**

機能を売るプロダクトではありません。「承認なし・自律分散・コンテキストを流通させる経営」というスタイルを、ツールの形で届けます。競合はHR SaaSや経費精算ツールではなく、**経営コンサルティング + それを実装するソフトウェア**の組み合わせです。

## ターゲット企業

### 理想の顧客像

Open Handbookを読んで「うちもやりたい」と思う企業です。機能比較で選ばれるのではなく、**経営思想への共感**が出発点になります。

| 属性 | 条件 |
|---|---|
| **社員数** | 30〜200人。ツールなしでも回る規模（10人以下）は対象外。意思決定が遅い大企業（500人以上）も初期ターゲットではない |
| **決裁者** | 代表、またはカルチャーに決裁権を持つ経営層。「ティール組織」「自律型組織」「透明性のある経営」に関心がある |
| **導入動機** | 機能の課題（「経費精算が面倒」）ではなく、組織の課題（「管理が増えてきた」「創業期のカルチャーが薄まってきた」） |
| **ITリテラシー** | 中〜高。Slack / Notion / GitHub 等を日常的に使っている組織 |

### 一次ターゲット

| セグメント | 特徴 | 刺さるポイント |
|---|---|---|
| **成長期のテック企業** | SaaS、受託開発、スタートアップ。30-150人。急成長で「創業時の空気が薄まってきた」と感じている | 管理を増やさずにスケールする仕組みとしてのTanden Toolkit。エンジニアが多くツール導入のハードルが低い |
| **クリエイティブ・専門職ファーム** | デザインファーム、コンサルティング、広告制作。20-80人。個人の専門性が価値の源泉 | 管理型では機能しない組織にとって「魂のカタチを活かす」がそのまま経営戦略になる |
| **世代交代を迎える中小企業** | 2代目・3代目が経営を引き継いだ、または引き継ぐ準備中。50-200人 | 「前の代のやり方を変えたい」がツール + コンサルティングのセットで実現できる。組織コンサルティングの需要が最も高い |

### 二次ターゲット

| セグメント | 特徴 | 刺さるポイント |
|---|---|---|
| **フルリモート企業** | 拠点を持たない、または分散チーム。非同期コミュニケーション + 透明性の仕組みが死活問題 | コンテキストが流通しないとリモート組織は崩壊する。Tanden Toolkitはその流通基盤 |
| **NPO・社会的企業** | ミッションドリブンで自律的な運営を志向。予算は限られるがカルチャーへの感度が高い | 思想への共感が深い。事例として発信力がある |

### ターゲットではない企業

- **10人以下** — ツールなしでも回る。導入コストに見合わない
- **500人以上の大企業** — 意思決定が遅く、セールスサイクルが12ヶ月以上。初期に追うべきではない
- **管理型を望む企業** — 承認フロー・階層型決裁を強化したい企業には思想が合わない。無理に売ると解約になる
- **機能で比較する企業** — 「勤怠SaaS 比較」で来た人はターゲットではない。コンテンツマーケでフィルタリングする

## ファネル

```
Open Handbook（無料公開）
  ↓ 思想に共感
カルチャーツール（Teian, Okuri, Tsuzuri, Meguri）
  ↓ 組織運営に踏み込む
ファイナンスツール（Warifu, Kodou, Motode, Karute）
  ↓ コミュニケーション基盤を移行
Uneri + Shiorie（チャット + AI）
  ↓ 完全移行
フルスイート + 組織コンサルティング
```

Open Handbookがファネルの入口です。ハンドブックを読んで「うちもやりたい」と思った企業がクライアントになります。広告費ゼロのコンテンツ・レッド・グロース。

## ロードマップ

### フェーズ1：PMF検証（0-12ヶ月）

Meibo + Teian + Okuri + Tsuzuri + Meguri をSlack連携で提供します。

- 開発コストが低く、PMF検証が速い
- Open Handbook経由で思想共感層を獲得
- Slack上でShiorieをボットとして提供（Uneriの前段階）
- 一次ターゲットのテック企業を中心に初期顧客を獲得
- 目標：10社導入、DAU70%以上、月次チャーン3%未満、MRR80万円

### フェーズ2：統合の価値を証明（12-24ヶ月）

Warifu + Kodou + Motode + Karute + Dandori を追加します。

- プロジェクトファイナンスの統合がキラーフィーチャー
- 「バラバラのSaaSよりTanden Toolkitで統合した方がいい」が明確になる
- 既存顧客のアップセル（ツール追加）でNRR120%以上を目指す
- 世代交代企業向けに組織コンサルティングの実績をつくる
- 目標：50社、MRR700万円、平均7ツール/社

### フェーズ3：プラットフォーム化（24-36ヶ月）

Uneri + Soubi + Noroshi を追加し、Slack依存を解消します。

- Shiorieのネイティブ統合で体験が劇的に変わる
- 既存顧客のSlack→Uneri移行を促す
- フルスイート導入率30%以上を目指す
- 目標：150社、MRR2,500万円（ARR3億円）

## 獲得戦略

| チャネル | 施策 |
|---|---|
| **Open Handbook** | SEO + SNS拡散。「自律型組織」「承認なし」「コンテキストマネジメント」等で検索流入 |
| **代表の発信** | 経営思想の発信（note、登壇、ポッドキャスト）。思想で惹きつけ、ツールで刈り取る |
| **事例公開** | 導入企業のBefore/Afterを詳細に公開。特にテック企業と世代交代企業の事例を優先 |
| **コミュニティ** | コンテキストマネジメントに関心のある経営者コミュニティ |
| **CDレーベルの伴走** | 導入初期の組織コンサルティングが口コミを生む |

**やらないこと：** マス広告、テレアポ、機能比較での競争。

## 競争優位性

| 優位性 | 説明 |
|---|---|
| **思想のロックイン** | 機能ではなく経営スタイルが刺さるので、他ツールに乗り換えにくい |
| **14ツールの深い統合** | データが途切れなく流れる。バラバラのSaaSでは再現不可能 |
| **Open Handbook** | 導入マニュアルが公開されており、CSコストが低い |
| **Shiorie** | ツール横断AIは既存SaaSが後付けでは実現しにくい |
| **Cloudflareスタック** | インフラコストが売上の1%未満。価格競争力が高い |

## 大企業戦略：パワースポットを置く

フェーズ3以降、大企業からの引き合いが発生した場合の戦略です。フェーズ1-2では追いません。

### 基本方針

大企業を「大口顧客」として追うのではなく、**大企業の中にパワースポットをつくる**アプローチを取ります。全社導入は狙わず、出島・新規事業・特定部門に限定して導入し、その部門の成功が社内に伝播するのを待ちます。

### 狙うべきエントリーポイント

| 部門タイプ | 特徴 | なぜ刺さるか |
|---|---|---|
| **新規事業部門・出島** | 本体と切り離された50-200人規模の組織 | 独自のカルチャーで回したい。本体のルールに縛られたくない |
| **CxO直轄の変革チーム** | 経営者が思想に共感し、トップダウンで実証を仕掛ける | 決裁が速い。成功すれば社内展開の起点になる |
| **M&A後の統合チーム** | 買収した小規模チームに文化を設計する必要がある | ゼロからカルチャーを入れられる。既存文化との衝突が少ない |

### やること・やらないこと

| やること | やらないこと |
|---|---|
| 部門単位の導入（50-300人） | 全社一括導入の提案 |
| 部門内の成功事例を社内コンテンツ化する支援 | 社内政治への介入 |
| 組織コンサルティングとのセット提供 | ツール単体の機能売り |
| 1社あたりARR上限を設定（全体の15%以内） | 1社依存になる契約 |

### 拡散メカニズム

導入部門が成果を出すと、社内で「あの部門、何か違うことやってるらしい」と噂になります。Tandenが営業するのではなく、**導入部門の成功事例が社内版Open Handbookの役割を果たす**ことで自然に広がります。

```
出島部門に導入（50人）
  ↓ 成果が見える化される
隣の部門が興味を持つ
  ↓ 導入部門が社内で布教（Tandenではなく顧客自身が）
2-3部門に拡大（200人）
  ↓ 経営層が全社検討を始める
全社導入の検討（1,000人+）
```

この拡散が起きるかどうかは、導入部門の体験の質にかかっています。だからこそ、最初の部門に対してCDレーベルが組織コンサルティングを含めて深く伴走することが重要です。

### 収益インパクト

| シナリオ | 社数 | 平均人数 | 年額 |
|---|---|---|---|
| 部門導入のみ | 5社 | 150人 | ¥5,000万 |
| 一部全社展開 | 2社 | 500人 | ¥6,600万 |
| **合計** | | | **¥1.2億** |

フェーズ3のARR ¥3億に加えて ¥1-2億の上乗せが見込めます。ただし、大企業対応のためにCDレーベルの専任バンドを組む必要があり、人件費も増加します。利益率を維持するには、組織コンサルティングの単価を十分に取ることが前提です。

## M&A戦略：クライアントをレーベルに迎える

Tanden Toolkitのクライアント企業を買収し、Tandenの新しい[レーベル](/docs/org-design/label)として迎え入れる成長戦略です。

### なぜこれが機能するか

M&Aの失敗原因は「カルチャーの衝突」と「システム統合の負荷」です。Tanden Toolkitのクライアントを買収する場合、この2つが**買収前に解決済み**です。

| 通常のM&A | Tandenクライアントの買収 |
|---|---|
| カルチャー統合に1-2年 | 買収前からTandenの思想で経営している |
| システム統合に半年-1年 | 同じTanden Toolkit上で稼働済み |
| 従業員の離反リスク | 思想に共感して使っているため抵抗が少ない |
| デューデリジェンスに数ヶ月 | Karuteで組織の健康状態をリアルタイムで把握済み |
| PMI（統合作業）が重い | レーベルとして迎え入れるだけ。自律性を維持したまま統合 |

### 買収プロセス

```
Tanden Toolkitクライアント（フルスイート導入済み）
  ↓ Karuteで組織の健全性を継続的に観察
  ↓ 事業シナジーの発見（CDレーベルの伴走を通じて）
買収提案
  ↓ デューデリジェンス（Karute + Warifuデータで大半が完了）
レーベルとして統合
  ↓ 旧経営者がレーベル主宰に就任
  ↓ Warifuコード体系をTanden全社に接続
  ↓ ボーナスがレーベルP&Lに連動
運営開始（Day 1から通常運転）
```

### 買収対象の条件

| 条件 | 理由 |
|---|---|
| フルスイート導入済み | システム統合コストをゼロにするため |
| 導入後12ヶ月以上 | 思想が定着していることを確認するため |
| 30-100人規模 | レーベルとして適切なサイズ |
| Tandenの既存レーベルとシナジーがある | 事業的な意味がなければ買収しない |
| 旧経営者がレーベル主宰として残る意思がある | 人とカルチャーごと迎え入れるため |

### Tandenらしさ

この戦略の本質は「買収」よりも「仲間になる」に近いです。

- 同じ思想で、同じツールで、同じ経営スタイルで動いている会社が、同じ屋根の下に入る
- レーベルの自律性は維持される。Tandenに「吸収」されるのではなく、[レーベル](/docs/org-design/label)として独自の美学を持ち続ける
- 結果として、Tandenは**多様な事業領域を持つレーベル連合体**として成長する
- ミッションの「社会にパワースポットを増やす」が、パワースポットを自社に取り込む形で実現する

## リスクと対策

| リスク | 対策 |
|---|---|
| 市場が狭い | 「自律型組織」に限定せず「透明性のある経営」まで広げれば対象は増える。世代交代企業は潜在市場が大きい |
| 14ツールの開発が重い | フェーズ分割で最小限から。カルチャーツールは開発が軽い |
| Slack移行ハードル | フェーズ1-2はSlack共存。Uneriは十分な価値が溜まってから |
| 文化が合わず解約 | 導入前にOpen Handbookで思想の合意を取り、ミスマッチを減らす |
| 大企業の全社導入要求 | 初期は断る。部門導入から始め、拡散を待つ。1社依存（ARR15%超）を避ける |
| 大企業向けカスタマイズ要求 | 「承認フローを追加して」等、思想と矛盾する要件は断る。Tandenの思想に合意した上での導入が前提 |
