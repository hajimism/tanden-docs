## 概要

**組織カルテ。** [Tanden Toolkit](/docs/tools)の全プロダクトからデータを集約し、会社の健康状態を可視化するダッシュボードです。財務だけでなく、関係性・学び・自律の循環まで映し出します。

このツールが伝えるコンテキストは**「会社は今こういう状態だ」**です。管理者向けのレポートではなく、全社員が[コンテキストを受け取る](/docs/culture/values/context)ための数字です。各自が自律的に判断するための材料を届けます。

全指標は全社員に公開されています。

## カスタマイズ可能なダッシュボード

Karuteは固定のレイアウトではなく、**自分で組めるダッシュボード**です。

- 利用可能な指標（後述）から、自分が見たいものを選んでダッシュボードを構成できます
- スコープも自由に切り替えられます（全社 / レーベル / フォルダ / コード / 個人）
- 構成は保存でき、複数のビューを切り替えられます

### シオリエによるアレンジ

[シオリエ](/docs/tools/shiorie)がダッシュボードの構成を提案します。

- 初回利用時、ロールや関わっているプロジェクトに応じたおすすめビューを自動生成します
- 「レーベル主宰向けのビュー組んで」「自分のパフォーマンスが見えるやつ」と頼めばアレンジしてくれます
- 指標に異常値が出たとき、関連する指標をまとめたビューを提案することもあります

### アラート

指標に異常値や注目すべき変化が現れた場合、シオリエがSlackで通知します。自分のダッシュボードに含まれる指標が対象です。

## 利用可能な指標

### 財務

| 指標 | データ取得元 |
|---|---|
| 売上 | Warifu（Revenue型コードの売上集計） |
| 利益率 | Warifu（Revenue型コードの売上 − コスト） |
| 生産性（粗利÷社員数） | Warifu + Meibo |

### プロジェクト

| 指標 | データ取得元 |
|---|---|
| Revenue型コードの利益率（個別・レーベル集計） | Warifu + Kodou + Motode |
| Budget型コードの消化率 | Warifu + Kodou + Motode |
| 稼働中コード数（レーベル別） | Warifu |
| コード別の人件費 vs 経費の内訳 | Kodou + Motode |
| タグ別の集計（フォルダ横断） | Warifu + Kodou + Motode |

### 組織

| 指標 | データ取得元 |
|---|---|
| 社員数（推移） | Meibo |
| 入社数・退社数（月次） | Meibo |
| 平均在籍期間 | Meibo |

### 関係性

| 指標 | データ取得元 |
|---|---|
| Okuriの総数（月次） | Okuri |
| 一人あたりOkuri送受信数 | Okuri + Meibo |
| Okuri未送信者率（孤立の兆候） | Okuri + Meibo |
| マイコーチ設定率 | Meibo Connections |

### 学び

| 指標 | データ取得元 |
|---|---|
| ステッカー流通量（月次） | Meguri |
| Meguriポイント消費率（貯め込まず使えているか） | Meguri |
| Tsuzuri外部公開数 | Tsuzuri |
| 書籍・カンファレンス投資総額 | Motode + Meguri |

### 自律

| 指標 | データ取得元 |
|---|---|
| テイアン起票数（月次） | Teian |
| テイアン平均レビュー参加人数（巻き込み力） | Teian |
| Noroshi消化率（掲載から1ヶ月以内に人が決まった割合） | Noroshi |
| Motode事後レビュー指摘率（低いほど自律的判断の精度が高い） | Motode |

### 日常

| 指標 | データ取得元 |
|---|---|
| Tsuzuri投稿率（毎日書いている人の割合） | Tsuzuri |
| Tsuzuri空欄率（高すぎると力が枯れている兆候） | Tsuzuri |
| 気分絵文字の分布（組織全体のムードマップ） | Tsuzuri |
| 対面全社会の参加率 | カレンダー / 手動 |

## シオリエ連携

[シオリエ](/docs/tools/shiorie)に話しかけるとKaruteの指標を引き出せます。

- 「今月の売上は？」→ 財務指標を参照
- 「PJT-ACMQ1の利益率は？」→ コード単位の収支を参照
- 「Acme案件フォルダの全体収支は？」→ フォルダ単位の集計を参照
- 「#フロントエンド タグの全コスト」→ タグでフォルダ横断集計
- 「組織の健康状態教えて」→ 各カテゴリの指標をサマリー
- 「レーベル主宰向けのダッシュボード組んで」→ ビューを提案・構成
- 「最近の異常値ある？」→ 注目すべき変化をピックアップ
