## 概要

**プロジェクト管理。** [Tanden Toolkit](/docs/tools)の一部として、[Shiorie](/docs/tools/shiorie)によるAI自動更新を中心に据えたライトウェイトなプロジェクト管理ツールです。[Warifu](/docs/tools/warifu)・[Kodou](/docs/tools/kodou)・[Meibo](/docs/tools/meibo)・[Uneri](/docs/tools/uneri)と深く統合されています。

このツールが伝えるコンテキストは**「段取りは生き物だ」**です。計画は立てた瞬間から変わります。会議の発言、メンバーの稼働状況、予算の消化——あらゆるコンテキストを自動で吸い上げ、段取りを常に最新に保ちます。手動でガントチャートを更新する時代は終わりです。

## なぜJira/Backlogではないか

| | Jira / Backlog | Dandori |
|---|---|---|
| 設計思想 | 網羅的な機能、重い | 最小限のUI + Shiorieが補完。軽い |
| 更新 | 人間が手動で更新 | Shiorieが会議や日報から自動更新 |
| 遅延予測 | ステータスの集計のみ | メンバーの有給・別案件・稼働実績を加味してAIが予測 |
| コスト | タスク管理とコスト管理が別 | タスクがWarifuコードに紐づき、原価が自動計算 |
| メンバー理解 | プロフィール程度 | Meiboの魂のカタチ・スキル・稼働状況を参照 |
| コミュニケーション | 別ツール（Slack等） | Uneriのハドル文字起こしから直接タスク更新 |

## 使い方

Dandoriの画面はシンプルです。タスク・マイルストーン・アサイン・期間を管理するだけ。複雑なフィールドやワークフロー設定はありません。足りない部分はShiorieが文脈から補完します。

- タスクの作成・編集は手動でもShiorie経由でもできます
- 「朝会の内容で更新して」とShiorieに頼めば、ハドルの文字起こしからタスクを更新します
- ガントチャートを人間が手動で引き直す必要はありません

## Shiorie による自動更新

DandoriにおけるShiorieの役割は**段取りを最新に保つこと**です。

### Uneriのハドルから更新

[Uneri](/docs/tools/uneri)のハドル（朝会・定例等）の自動文字起こしをShiorieが解析し、タスクの更新を提案します。

- 「ヒアリング終わりました」→ タスクを完了に
- 「来週火曜有給です」→ 影響するタスクの日程を再計算
- 「別案件が水木で入りそう」→ 稼働可能時間を調整して遅延予測

提案はUneriのスレッドに投稿されます。確認して承認すればDandoriに反映されます。

### Kodouからの実績反映

[Kodou](/docs/tools/kodou)の工数データから、計画と実績のズレを検出します。

- 予定5日のタスクに既に4日チャージされているが進捗50% → 遅延アラート
- あるメンバーのSLFコード比率が高い → 手が空いている可能性を示唆

### Tsuzuriからの状況把握

[Tsuzuri](/docs/tools/tsudzuri)の日報から、メンバーの状態をShiorieが把握します。

- 気分絵文字が低調 → 負荷の再配分を提案
- 「設計で詰まっている」と書かれている → 遅延リスクとしてフラグ

### 遅延予測

Shiorieは以下を総合して遅延予測を行います。

- タスクの依存関係と残日数
- メンバーごとの稼働可能時間（Kodouの実績 + 有給 + 別案件）
- 過去の類似タスクの実績（見積もり精度の学習）
- Tsuzuriから読み取れるメンバーの状態

```
Shiorie: PJT-ACMQ1の遅延予測です。
        
        ⚠️ 要件書ドラフト（鈴木さん）
        - 予定: 3/1-3/7（5日）
        - 3/4-5に別案件（PJT-XYZ）稼働予定
        - 実質稼働3日に短縮 → 3/10完了見込み
        
        → レビュー開始が3/11→3/13にずれます。
          設計フェーズに影響が出る可能性があります。
        
        対策案:
        1. 山田さん（3/1-3/7空き）にドラフトを分担
        2. レビューを1日に短縮（非同期レビュー）
```

## Warifuコード連携

Dandoriのプロジェクトは[Warifuコード](/docs/tools/warifu)に紐づきます。

- タスクに投入された工数（Kodou経由）がコードにチャージされる
- タスク単位のコスト内訳を表示
- 予算消化率とタスク進捗を重ねて「このペースで予算は足りるか」をShiorieが予測

## Meibo連携

タスクのアサイン時にShiorieが[Meibo](/docs/tools/meibo)を参照します。

- 「フロントエンドが得意な人いる？」→ スキルタグで候補を提案
- アサイン候補の現在の稼働状況（Kodou）を表示
- 魂のカタチを参照して「この人にはこういう仕事の渡し方が合いそう」を提案

## シオリエ連携

[シオリエ](/docs/tools/shiorie)に話しかけるとDandoriの操作ができます。

- 「PJT-ACMQ1の進捗は？」→ 状況サマリー
- 「遅れそうなタスクある？」→ 遅延予測を表示
- 「来週の段取り見せて」→ 直近のタスク・マイルストーン一覧
- 「朝会の内容でタスク更新して」→ ハドル文字起こしからタスク更新を提案
- 「山田さんの来週の稼働どれくらい空いてる？」→ Kodou + Dandoriから算出
