## 概要

**プロジェクトファイナンスの基盤。** [Tanden Toolkit](/docs/tools)の一部として、予算の委譲とWarifuコードの発行・管理を行います。[Kodou](/docs/tools/kodou)・[Motode](/docs/tools/motode)・[Karute](/docs/tools/karute)と深く統合されています。

このツールが伝えるコンテキストは**「お金は信頼とともに渡される」**です。割符は古来、木片を二つに割って双方が持ち合い、合わせることで本物と証明する仕組みでした。予算を「割って渡す」行為に、委ねる側の信頼と受け取る側の責任が宿ります。

## Warifuフォルダとコード

Warifuは**フォルダ**と**コード**で構成されます。ファイルシステムと同じ構造です。

- **Warifuフォルダ**：コードや子フォルダを束ねる入れ子可能な箱。誰でも作れます。フォルダ自体は予算を持たず、配下の全コードの集計ビューとして機能します。任意のタイミングでアーカイブ可能です
- **Warifuコード**：実際に予算がつく最小単位。誰でも発行できますが、予算をつけるのは予算を持っている人だけです

```
レーベル「飲食」/
  ├── Acme案件/
  │    ├── Q1/
  │    │    ├── PJT-ACMQ1（コード）
  │    │    └── OPS-ACM0Q1（コード、Q1運営費）
  │    └── Q2/
  │         └── PJT-ACMQ2（コード）
  ├── 新業態実験/
  │    └── PJT-NEW001（コード）
  └── レーベル運営/
       └── OPS-LAB001（コード）
```

フォルダの入れ子に深さの制限はありません。レーベル → クライアント → プロジェクト → フェーズのような自然な階層がそのまま表現できます。どのフォルダを開いても、配下の全コードの予算・消化・利益の集計が見えます。[Karute](/docs/tools/karute)では任意のフォルダ単位で収支を集計できます。

### Warifuフォルダを作る

何かを始めるとき、最初にやるのは**Warifuフォルダを作ること**です。

- 新しいクライアント案件が決まった → フォルダを作る
- 社内プロジェクトを立ち上げたい → フォルダを作る
- 採用活動をまとめたい → フォルダを作る

フォルダ作成に承認は不要です。誰でも、いつでも作れます。フォルダがあることで「ここに何かが始まろうとしている」というコンテキストが全社に見えます。予算がついていなくても、フォルダとコードが存在すること自体が意思表示です。

### Warifuコード

コードの形式は `XXX-YYYYYY` です。

- **XXX**：prefix。プロジェクトのタイプとクラス（Revenue / Budget）を示します
- **YYYYYY**：英数字6桁。ユニークであれば自由に決められます

| クラス | prefix例 | 意味 |
|---|---|---|
| Revenue（売上型） | `PJT` | クライアント案件。売上と利益率を追跡 |
| Budget（投資型） | `OPS`, `WCL`, `LRN`, `HIR`, `STD`, `SLF` | 社内投資。予算の消化率を追跡 |

### コードの属性

| 項目 | 内容 |
|---|---|
| 責任者 | 1コードにつき1人。必須。責任者本人がいつでも他者に付け替え可能。社長は任意のコードの責任者を強制変更できます |
| 予算額 | 予算を持つ人が割り当て。未割り当て（予算待ち）のコードも存在できる |
| 有効期間 | 開始日・終了日。Revenue型は四半期単位を推奨 |
| 所属フォルダ | 必須。どのフォルダに属するか |
| タグ | 任意。複数可。フォルダを横断した集計に使用 |
| クラス | prefixで自動決定 |

### タグによる横断集計

コードは1つのフォルダにしか属せませんが、タグは複数つけられます。フォルダがツリー構造（組織・案件の階層）を表すのに対して、タグはツリーを横断する切り口を提供します。

- `#フロントエンド` — 全レーベルのフロントエンド案件を横断集計
- `#新規` — 既存案件と新規案件のコスト比較
- `#採用` — 採用関連コストをレーベル横断で把握

タグは既存タグから選択するか、新しいタグを自由に作成できます。[シオリエ](/docs/tools/shiorie)が定期的にタグの表記ゆれを検出し、統合を提案します。

[Karute](/docs/tools/karute)でタグ単位の集計が見られます。フォルダが「どこに属するか」、タグが「何に関するか」です。

予算の考え方の全体像は[予算の考え方](/docs/finance/budget)を参照してください。

## 透明性

全コード・全フォルダ・全予算は常に全社員に公開されています。誰がどのコードにいくらの予算を持ち、どれだけ消化しているかは全員が見える状態です。この透明性が、承認なしでも健全な判断を支えます。

## 予算の委譲

予算は人から人へ委譲できます。

- 委譲は誰でもできます。自分が持つ予算の範囲内で自由に渡せます
- 受け取った予算でコードを発行するか、さらに他者に委譲するかは受け取った人の判断です
- 委譲のトランザクション（誰が誰に、いつ、いくら渡したか）は全て記録されます

**社長の特権：** 社長のみ、任意のコードや個人から予算を強制回収できます。ガバナンスの最後の砦です。

## コードのライフサイクル

| 状態 | 説明 |
|---|---|
| **発行（予算待ち）** | 誰でもコードを作成できる。予算がまだついていない状態。企画段階で先にコードを切っておける |
| **稼働中** | 予算がつき、人件費（[Kodou](/docs/tools/kodou)経由）と経費（[Motode](/docs/tools/motode)経由）がチャージされる |
| **フリーズ** | 責任者が凍結。新規チャージ不可、精算中の経費は継続 |
| **完了** | 有効期間満了 or 責任者が手動クローズ。残予算は委譲元に戻る |

### 予算の追加

コードの予算が不足した場合、追加予算を[テイアン](/docs/systems/teian)で獲得します。テイアン先は自由です。

## コードへのチャージ

Warifuコードには2種類のコストがチャージされます。

| 種類 | 仕組み | 連携ツール |
|---|---|---|
| **人件費** | 社内単価 × 投入時間 | [Kodou](/docs/tools/kodou) |
| **経費** | 交通費・飲食費・備品等の実費 | [Motode](/docs/tools/motode) |

### 予算超過の扱い

チャージは予算残高に関係なく常に記録できます。予算を超過してもブロックされません。予算ゼロ（予算待ち）のコードにもチャージ可能です。

- 予算超過・赤字は[Karute](/docs/tools/karute)で可視化され、責任者に通知されます
- 超過が続く場合、責任者が追加予算を[テイアン](/docs/systems/teian)するか、コードをフリーズするか判断します
- 実態を正確に記録することが最優先です。「予算がないから記録しない」はダサいです

## Revenue型コードの収支

Revenue型コードには売上を記録できます。責任者が請求のタイミングで売上を入力します。売上 − コスト = 利益で、利益率が自動算出されます。[Karute](/docs/tools/karute)でコード単位・レーベル単位の収支を確認できます。

## 自動発行されるコード

### セルフマネジメントコード（SLF）

入社時に全員に `SLF-XXXXXX` コードが自動発行されます。有効期限はありません。

- 特定のプロジェクトに属さない時間のチャージ先です
- 思考・インプット・1on1・社内の相談対応など、どのコードにも乗らない時間はここに刻みます
- 予算はレーベルまたは全社から割り当てられます（従業員を抱えるコストの一部）

### 全社イベントコード

全社会・[マチネ](/docs/work-style/matinee)など全員参加のイベントには、イベントごとにコードが発行されます。参加者は該当コードに鼓動を刻みます。

### WCLコード（ウェルカムランチ）

[オンボーディングクエスト](/docs/onboarding/quests)を達成すると、本人用の `WCL-XXXXXX` コードが自動発行されます。

- 予算：¥12,000
- 用途：食費であれば自由（目安1食¥1,500だが、使い方は本人次第）
- この予算で最大の効果を出す使い方を自分でデザインしてください
- 経費精算は[Motode](/docs/tools/motode)から、コードを指定して行います

詳しくは[ウェルカムランチ](/docs/systems/welcome-lunch)を参照してください。

## シオリエ連携

[シオリエ](/docs/tools/shiorie)に話しかけるとWarifuの情報を引き出せます。

- 「PJT-ACMQ1の残予算は？」→ コードの予算状況を参照
- 「Acme案件の全体収支は？」→ フォルダ単位の集計を表示
- 「自分が関わってるコード一覧」→ アクティブなコード一覧
- 「新しいフォルダ作りたい」→ フォルダ作成フローを起動
- 「新しいコード発行したい」→ コード発行フローを起動
